# Calendar2026_Csharp_app
UI/UX学年発表会【Calendar2026】

※わかりずらいため大変恐縮ですが、「Form3.cs→Form1.cs→Form2.cs」の流れでコードを読むことができます。2025年6月に制作しました。

------------------------------------------------------------------------------------------------------------------------------

「概要」

本プロジェクトは日々の業務に疲れた人へ、癒しを届けることを目的に制作しました。

このミッションを達成するために、全て自作の画像を使い、情景にあったBGMを採用しています。

※BGMは「DOVA-SYNDROME YouTube Official」様で公開されているものを使用させていただいております。

------------------------------------------------------------------------------------------------------------------------------

「システムの内容」

本アプリの主な機能は2026年のカレンダーです。

2026年でカレンダーを制作した理由は、当時2025年の6月で、すでに半年経っていたためでした。

------------------------------------------------------------------------------------------------------------------------------

「ターゲット層」

疲れがたまり、癒しを求めている20～30代。

------------------------------------------------------------------------------------------------------------------------------

「作成背景」

もともとは学内の制作課題として「四季を表示する」というテーマで、UI/UXが評価される発表会でした。

そこで私は、ただ見やすく、操作しやすいだけではなく、さらに感動を届けられるようなものを作ろうと思いました。

デザインを学んでいた身なので、これは負けられないと思い、誠心誠意で制作にあたる期間でした。

※余談になりますが、イラストは1枚あたり1週間程度で制作しています。

「機能一覧」

・カレンダー機能：シンプルで丸みのある数字が特徴です。祝日はもちろん、その月の行事まで記載されています。

・祝日機能：ホーム画面で選択されている月の祝日を詳細に教えてくれる機能です。私なりに解釈し記載しました。

・チュートリアル機能：インコが次への操作方法を教えてくれるので、迷わずに操作することができます。
また、チュートリアルはアプリ内で何度でも再表示することができます。

・音楽操作機能：簡易的ではありますが、ホーム曲を選択する機能と音楽のみミュートにする機能がございます。
効果音もミュートにしたい場合はパソコンのミュート機能を使うと解決するので制作しませんでした。

「工夫した点」

・楽しく貯金やタスクを達成してもらうために、ミッション機能やキャラクター会話機能、音楽機能などを取り入れました。
また、Wishlist機能で自身のやりたいこと、ほしいものを言語化し目的意欲の向上を図っています。

・できるだけボタン回数を減らすために、ユーザーが自然な動作(選択を解除するなど)をするときにデータベースへ保存していました。
また、本アプリは毎日することを推奨しているため、スタート画面をなくし起動を早くしていたり、デイリーやミッションはホーム画面からワンタップで達成できるようにしています。
加えてキャラクターのセリフも一定時間ごとに自動で変わるようにしています。

・項目を分けることでタスクごとの種類をわかりやすくしています。若い人向けのため、全体的に色は明るく彩度が薄いものを使用するように心がけていました。

------------------------------------------------------------------------------------------------------------------------------

「課題・今後の改善」

・PC上でないと起動できないため毎日開くのが困難な点が課題だと感じました。そのため、現在Web開発として再度制作中です。

・初めて触る人がどんなアプリかイメージしずらいことが課題だと感じました。ハテナマークから説明書が表示されますが、よりシンプルに開発することで理解
しやすくしようと思います。具体的には入力する際に選択式のモードで区別します。その後並び替えでタスクの種類をわけ、わかりやすさを維持しようと考えています。
